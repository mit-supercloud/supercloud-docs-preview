    Databases

    Databases

    The MIT Supercloud allows users to launch their own databases through the database portal. The portal is located at:

    https://txe1-portal.mit.edu/db/dbstatus.php

    This page requires you to authenticate into the portal. From here you will see all the databases you have access to. We currently support Accumulo (1.5.0, 1.6.0, 1.7.0, and 1.8.0) and PostgreSQL databases through our dynamic database capability.

    To start up a database instance, press "Start". You can stop it by clicking on the "Stop" button, and can checkpoint a stopped database as well. Clicking on "View Info" for Accumulo databases will take you to the Accumulo Monitoring page, where you can view ingest/query plots and current tables.

    For most of your database uses, you are unlikely to need more information than what the above web portal database interfaces provide. However, in some cases you may need to get more information. For each of your databases, much more instance information is available on the central storage at /home/gridsan/groups/databases/<database-name>/.

    If you need an instance created and cannot use one of those already available, contact us. Let us know:

    • The type of database you need
    • What you are using it for
    • What it should be called
    • Who should have access to the database

    Additionally, please let us know if there are any special configurations you need.

    There are many ways to insert and query data. While this cannot be done from the portal page, you can query and insert data from any of the nodes on the MIT Supercloud system. We recommend using D4M, which has been installed and pre-configured to work on our system with very little effort. For more information about databases and how to use them with D4M, take a look at the Advanced Database Technologies course on our online course platform. This course contains a good introduction on how to set up a data pipeline, including parsing, ingesting, and querying data for Accumulo.

    Working with Accumulo Databases on SuperCloud

    Once an Accumulo database has been created, you will need to get a variety of metadata about the database including the password to access the database. This is stored as database metadata information on the central storage at /home/gridsan/groups/databases/<database-name>/. In that directory, you will find a number of files including one that contains the user password (accumulo_user_password.txt).

    Most of these files are read-only because the dynamic database management system requires exclusive access to them to manage the databases. However, the contents can be read by your code. For instance, the user password will change each time you launch the database onto the cluster. Therefore, reading the password from the file to insert into your database access code would be an efficient way to work with that dynamicism.

    Connecting to an Accumulo Instance Programmatically

    Here is a sample Java code snippet that is used to programmatically connect to an Accumulo instance called my-instance-name:

    import java.io.File;
import java.io.FileFilter;
import java.io.BufferedReader;
import java.io.FileNotFoundException;
import java.io.FileReader;
import org.apache.accumulo.core.client.*;
import org.apache.accumulo.core.client.security.tokens.PasswordToken;
import org.apache.accumulo.core.client.ZooKeeperInstance;
import org.apache.zookeeper.ZooKeeper;
...
String instanceName = "my-instance-name";
String pwdFile = "/home/gridsan/groups/databases/" + instanceName + "/accumulo_user_password.txt";
File file = new File(pwdFile);
try (BufferedReader is = new BufferedReader(new FileReader(file))) {
    String password = is.readLine();
    token = new PasswordToken(password.getBytes());
} catch (FileNotFoundException e) {
    System.out.println("Cannot find accumulo_user_password.txt for instance " + instanceName + " " + e);
    throw new RuntimeException(e);
} catch (IOException e) {
    System.out.println("Problem reading accumulo_user_password.txt for instance " + instanceName + " " + e);
    throw new RuntimeException(e);
}
String userName = "AccumuloUser";
String zooServers = instanceName + ".cloud.llgrid.ll.mit.edu:2181";
System.out.println("zooServer = <" + zooServers + ">");
ZooKeeperInstance instance = new ZooKeeperInstance(instanceName, zooServers);
Connector connector = instance.getConnector(userName, token);

    Working with PostgreSQL Databases on SuperCloud

    Most SQL databases including PostgreSQL trade high performance for transaction atomicity and data consistency. Hence, there are situations in which the use of a SQL database is necessary.

    On SuperCloud, PostgreSQL is available within our dynamic database capability. You will need to request that a PostgreSQL dynamic database be created for you by emailing the SuperCloud team.

    Once the PostgreSQL database has been created, you will need to get a variety of metadata about the database including the username and password to access the database. This is stored as database metadata information on the central storage at /home/gridsan/groups/databases/<database-name>/. In that directory, you will find a number of files including one that contains the username (dbusername) and the user password (postgresql_user_password.txt).

    Most of these files are read-only because the dynamic database management system requires exclusive access to them to manage the databases. However, the contents can be read by your code. For instance, the user password will change each time you launch the database onto the cluster. Therefore, reading the password from the file to insert into your database access code would be an efficient way to work with that dynamicism.
